1. Потоки ввода/вывода. Создать файл, используя команду echo. Используя команду cat, 
прочитать содержимое каталога etc, ошибки перенаправить в отдельный файл.

--------Создание файла чере echo-----------
$ echo "new file" >> test.txt
--------Запись ошибок при чтении каталога etc без вывода в терминал остальной инфы--
$ cat /etc/* 2>errorLog.log 1>/dev/null

===========================================================================
2. Конвейер (pipeline). Использовать команду cut на вывод длинного списка каталога, чтобы 
отобразить только права доступа к файлам. Затем отправить в конвейере этот вывод на 
sort и uniq, чтобы отфильтровать все повторяющиеся строки.

$ ls /etc -l | cut -b 1-10 | sort | uniq 1> test.txt

===========================================================================
3. Управление процессами.) Изменить конфигурационный файл службы SSH: /etc/ssh/sshd_config, отключив 
аутентификацию по паролю PasswordAuthentication no. Выполните рестарт службы 
systemctl restart sshd (service sshd restart), верните аутентификацию по паролю, 
выполните reload службы systemctl reload sshd (services sshd reload). 
В чём различие между действиями restart и reload? Создайте файл при помощи команды 
cat > file_name, напишите текст и завершите комбинацией ctrl+d. Какой сигнал передадим 
процессу?

$ sudo systemctl restart sshd
$ sudo systemctl reload sshd
--Как заметил, при reload процесс не останавлиется, вероятно, лишь перечитывается файл конфигурации--
--При restart процесс останавливается и начинается заново--
--Это видно по изменению номера процесса. В первом случае он не меняется, во втором присвается новый номер--

$ cat > file_name.txt
--После этой команды заметил, что на время ввода текста был выделен процесс ожидания ввода текста--
--Нажатием crtl+d говорит о завершении файла - команда EndOfFile

===========================================================================
4. Сигналы процессам. Запустите mc. Используя ps, найдите PID процесса, 
завершите процесс, передав ему сигнал 9.

$ mc
$ ps -elf | grep mc
--Находим процесс mc - 73467 {да, компьютер давно работает} и просим его закрыться---
$ kill -n 9 73467
--Проверим, что такого процесса больше нет---
$ ps -elf | grep mc
